1. DESCRIPCION GENERAL
--------------------------------------------------------------------------------

Se realizo el preprocesamiento de imagenes de resonancia magnetica funcional
(fMRI) en estado de reposo de pacientes post-ACV, utilizando fMRIPrep v24.1.1
ejecutado mediante Docker en un sistema Windows 10/11. Este enfoque permite
ejecutar herramientas disenadas para Linux en cualquier sistema operativo,
garantizando la reproducibilidad del analisis.


2. DOCKER: EJECUCION MULTIPLATAFORMA
--------------------------------------------------------------------------------

fMRIPrep y sus dependencias (FreeSurfer, ANTs, FSL, Nipype) fueron desarrolladas
para sistemas Linux. Para ejecutarlas en Windows se utilizo Docker Desktop, una
plataforma de contenedores que empaqueta aplicaciones con todas sus dependencias
en un entorno aislado.

Un contenedor Docker funciona como un sistema Linux ligero dentro de Windows.
A diferencia de una maquina virtual completa, comparte el kernel del sistema
anfitirion, lo que lo hace mas eficiente en uso de memoria y procesamiento.

Cuando se ejecuta el comando "docker run", Docker:
  1. Crea un contenedor Linux temporal
  2. Monta las carpetas de Windows como volumenes accesibles
  3. Ejecuta fMRIPrep dentro del contenedor
  4. Escribe los resultados directamente en las carpetas de Windows
  5. Destruye el contenedor al finalizar (flag --rm)

Imagen utilizada:
  nipreps/fmriprep:24.1.1 (~15-20 GB)

Descarga (se realiza una sola vez):
  docker pull nipreps/fmriprep:24.1.1

Esta imagen contiene:
  - fMRIPrep (pipeline principal de preprocesamiento)
  - FreeSurfer (segmentacion y reconstruccion cortical)
  - ANTs (Advanced Normalization Tools, registro y normalizacion)
  - FSL (FMRIB Software Library, herramientas de analisis)
  - Nipype (motor de flujos de trabajo en Python)
  - Todas las librerias y configuraciones necesarias

El usuario no necesita instalar ninguna de estas herramientas individualmente.


3. LICENCIA DE FREESURFER
--------------------------------------------------------------------------------

FreeSurfer, desarrollado por el Martinos Center for Biomedical Imaging
(Harvard/MIT), requiere un archivo de licencia para ejecutarse. Esta licencia
es gratuita para uso academico.

Obtencion:
  1. Registrarse en: https://surfer.nmr.mgh.harvard.edu/registration.html
  2. Completar formulario con nombre, correo electronico e institucion
  3. Recibir por correo el archivo "license.txt"

Ubicacion en el proyecto:
  C:\Users\ASUS\Documents\OpenNeuro\license.txt

NOTA: El archivo de licencia es requerido incluso cuando se desactiva la
reconstruccion cortical de FreeSurfer (--fs-no-reconall), ya que fMRIPrep
valida su existencia al inicio de la ejecucion. No es necesario instalar
FreeSurfer por separado; ya esta incluido en la imagen Docker.


4. COMANDO DE EJECUCION
--------------------------------------------------------------------------------

Ejecutado desde PowerShell en Windows:

docker run --rm
    -v C:\Users\ASUS\Documents\OpenNeuro:/data:ro
    -v C:\Users\ASUS\Documents\OpenNeuro\derivatives:/out
    -v C:\Users\ASUS\Documents\OpenNeuro\work:/work
    -v C:\Users\ASUS\Documents\OpenNeuro\license.txt:/opt/freesurfer/license.txt:ro
    nipreps/fmriprep:24.1.1
    /data /out participant
    --participant-label 00
    --output-spaces MNI152NLin2009cAsym:res-2 anat
    --nprocs 8
    --mem-mb 16384
    --work-dir /work
    --skip_bids_validation
    --fs-no-reconall

Explicacion de parametros:

  --rm
    Elimina el contenedor Docker al finalizar para liberar espacio en disco.

  -v C:\...\OpenNeuro:/data:ro
    Monta el dataset BIDS como volumen de solo lectura (:ro) dentro del
    contenedor. Esto protege los datos originales de cualquier modificacion.

  -v C:\...\derivatives:/out
    Monta la carpeta de salida donde se guardaran los resultados.

  -v C:\...\work:/work
    Monta la carpeta para archivos intermedios temporales.

  -v license.txt:/opt/freesurfer/license.txt:ro
    Monta el archivo de licencia de FreeSurfer en la ruta que espera el
    software dentro del contenedor.

  --participant-label 00
    Indica que se procese unicamente el sujeto sub-00.

  --output-spaces MNI152NLin2009cAsym:res-2 anat
    Genera salidas en dos espacios:
      - MNI152NLin2009cAsym con resolucion de 2mm (espacio estandar)
      - Espacio nativo anatomico del sujeto (anat)

  --nprocs 8
    Utiliza 8 nucleos de CPU para procesamiento paralelo.

  --mem-mb 16384
    Limita el uso de memoria RAM a 16 GB.

  --skip_bids_validation
    Omite la validacion BIDS (el dataset ya fue validado previamente).

  --fs-no-reconall
    Desactiva la reconstruccion cortical completa de FreeSurfer. Esto reduce
    significativamente el tiempo de procesamiento (de ~10 horas a ~1-2 horas
    por sujeto) y es suficiente para analisis de conectividad funcional.

6. PASOS DEL PIPELINE fMRIPrep
--------------------------------------------------------------------------------

6.1 PREPROCESAMIENTO ANATOMICO (T1w)

  a) Correccion de inhomogeneidad de campo (N4):
     Corrige variaciones de intensidad en la imagen causadas por
     irregularidades del campo magnetico B1.

  b) Extraccion cerebral (brain extraction):
     Elimina el craneo, piel y otros tejidos no cerebrales,
     dejando unicamente el volumen cerebral.

  c) Segmentacion tisular:
     Clasifica cada voxel en tres tipos de tejido:
       - Sustancia gris (GM): cuerpos neuronales
       - Sustancia blanca (WM): axones mielinizados
       - Liquido cefalorraquideo (CSF)

  d) Normalizacion espacial:
     Registra la anatomia individual del sujeto al espacio estandar
     MNI152NLin2009cAsym, permitiendo comparaciones entre sujetos.

6.2 PREPROCESAMIENTO FUNCIONAL (BOLD)

  a) Generacion de imagen de referencia:
     Crea un volumen de referencia estable a partir de la serie temporal.

  b) Slice Timing Correction:
     Corrige las diferencias temporales entre los 41 cortes, que fueron
     adquiridos de forma intercalada (no simultanea) dentro de cada TR.

  c) Correccion de movimiento (Motion Correction):
     Estima 6 parametros de movimiento rigido (3 traslaciones en X, Y, Z
     y 3 rotaciones en X, Y, Z) y alinea todos los volumenes al volumen
     de referencia. Incluso movimientos de 1-2 mm pueden contaminar la
     senal BOLD.

  d) Co-registro funcional-anatomico:
     Alinea las imagenes funcionales (resolucion 3 mm) con la imagen
     anatomica T1w (resolucion 1 mm) del mismo sujeto.

  e) Normalizacion al espacio MNI:
     Aplica las transformaciones calculadas en el paso anatomico para
     llevar los datos funcionales al espacio estandar MNI.

  f) Generacion de confounds:
     Calcula multiples variables de confusion para uso posterior:
       - 6 parametros de movimiento y sus derivadas temporales
       - Componentes CompCor (senales de WM y CSF)
       - Framewise displacement (indice de movimiento entre volumenes)
       - Senales globales (cerebro, WM, CSF)

8. LIBRERIAS Y HERRAMIENTAS UTILIZADAS
--------------------------------------------------------------------------------

La imagen Docker de fMRIPrep empaqueta todas las herramientas necesarias para
el preprocesamiento. A continuacion se detallan las librerias principales y su
funcion dentro del pipeline.

8.1 HERRAMIENTAS DE NEUROIMAGEN (C/C++)

  ANTs (Advanced Normalization Tools)
    Registro no lineal, normalizacion al espacio MNI, correccion de
    inhomogeneidad de campo N4.

  FreeSurfer
    Segmentacion tisular, reconstruccion de superficies corticales.
    En este proyecto fue desactivado con --fs-no-reconall.

  FSL (FMRIB Software Library)
    Extraccion cerebral (BET), correccion de movimiento (MCFLIRT),
    slice timing correction.

  AFNI
    Correccion de despiking (eliminacion de picos de intensidad anomalos),
    herramientas auxiliares de preprocesamiento.

  Convert3D (C3D)
    Conversion y manipulacion de formatos de imagen medica.

8.2 LIBRERIAS PYTHON DEL PIPELINE

  Nipype
    Motor de flujos de trabajo. Conecta todas las herramientas en un pipeline
    reproducible y paralelizable.

  Nilearn
    Procesamiento de senales cerebrales, limpieza de confounds, manipulacion
    de imagenes NIfTI.

  Nibabel
    Lectura y escritura de archivos NIfTI, GIFTI y CIFTI.

  NumPy
    Operaciones numericas y manejo de matrices multidimensionales.

  SciPy
    Interpolacion, filtrado de senales, operaciones estadisticas.

  Pandas
    Manejo de tablas de confounds (archivos TSV).

  Matplotlib
    Generacion de figuras y graficos para los reportes de calidad.

  Scikit-learn
    Calculo de componentes CompCor mediante PCA (Analisis de Componentes
    Principales).

  TemplateFlow
    Descarga y gestion de templates cerebrales estandarizados (MNI152, etc.).

  SDCFlows
    Correccion de distorsiones espaciales por susceptibilidad magnetica.

  NiWorkflows
    Flujos de trabajo reutilizables para generacion de reportes visuales
    (HTML + SVG).

  SMRiPrep
    Sub-pipeline dedicado al preprocesamiento anatomico (T1w/T2w).

8.3 DIAGRAMA DE DEPENDENCIAS

  fMRIPrep (pipeline principal)
   |-- SMRiPrep --- Preprocesamiento anatomico
   |    |-- ANTs ---- Normalizacion, correccion N4
   |    |-- FreeSurfer -- Segmentacion (opcional)
   |    +-- FSL ---- Brain extraction (BET)
   |
   |-- Nipype ------ Motor de flujos de trabajo
   |
   |-- SDCFlows --- Correccion de distorsiones
   |
   |-- FSL -------- Motion correction (MCFLIRT), slice timing
   |-- AFNI ------- Despiking
   |
   |-- Nilearn ---- Limpieza de senal, confounds
   |-- Nibabel ---- Lectura/escritura de archivos NIfTI
   |-- TemplateFlow -- Templates MNI
   |
   +-- NiWorkflows -- Reportes HTML + figuras SVG

Todas estas herramientas estan preinstaladas en la imagen Docker
nipreps/fmriprep:24.1.1 (~15-20 GB). No es necesario instalar ninguna
de ellas manualmente.

10. REPRODUCIBILIDAD
--------------------------------------------------------------------------------

Para reproducir este preprocesamiento:

  1. Instalar Docker Desktop en cualquier sistema operativo
  2. Descargar la imagen: docker pull nipreps/fmriprep:24.1.1
  3. Obtener la licencia gratuita de FreeSurfer
  4. Ejecutar el comando documentado en la seccion 4 para cada sujeto

La imagen Docker garantiza que todas las versiones de software son identicas
independientemente del sistema operativo, asegurando reproducibilidad completa.
